## 1. 系统概述

### 1.1 系统简介

AI-OA是一款基于RuoYi框架的智能化OA系统，整合了OCR识别、n8n流程引擎和AI大模型能力。

### 1.2 主要功能

| 模块 | 功能 |
|------|------|
| 工作台 | 仪表盘、快捷操作、待办事项 |
| 审批中心 | 多级审批流程、审批历史 |
| 财务报销 | OCR发票识别、自动填单 |
| AI助手 | 智能问答、知识库检索 |
| 企业聊天 | 即时消息、群聊、文件传输 |
| 智能报表 | 周报/月报/年刊自动生成 |

### 1.3 系统入口

| 端 | 访问地址 |
|----|----------|
| PC Web | http://服务器IP |
| 移动端 | 鸿蒙/iOS/Android APP |
| API | http://服务器IP:8080 |

## 5. 财务报销

### 5.1 OCR发票识别

```bash
# 操作步骤：
# 1. 点击"拍照识别"或上传发票图片
# 2. 系统自动识别发票信息
# 3. 核对识别结果
# 4. 确认无误后提交报销
```

### 5.2 支持的发票类型

| 类型 | 说明 |
|------|------|
| 增值税专用发票 | 税率13%/9%/6% |
| 增值税普通发票 | 电子/纸质 |
| 出租车票 | 行程单对接 |
| 火车票/机票 | 行程单对接 |
| 餐饮发票 | 普票/专票 |

### 5.3 OCR置信度说明

| 置信度 | 显示 | 处理方式 |
|--------|------|----------|
| ≥85% | 绿色 ✅ | 可直接提交 |
| <85% | 黄色 ⚠️ | 需人工确认 |
| <60% | 红色 ❌ | 需重新上传 |

### 5.4 疑问单据处理

当OCR置信度<85%时：

1. 系统自动高亮可疑字段
2. 邮件通知审批人
3. 邮件通知提交人
4. 暂停自动审批流程
5. 人工确认后恢复

## 9. 知识库

### 9.1 知识库结构

```
知识库
├── 公司制度
│   ├── 人事制度
│   └── 财务制度
├── 产品文档
```

### 9.2 知识检索

| 方式 | 说明 |
|------|------|
| 关键词搜索 | 精准匹配 |

### 9.3 知识贡献

员工可以提交知识，经审核后入库：

1. 点击"📝 贡献知识"
2. 填写知识标题和内容
3. 选择分类和标签
4. 提交审核
5. 管理员审核通过后公开

## 附录A：常见问题

### A.1 登录问题

| 问题 | 解决方案 |
|------|----------|
| 忘记密码 | 联系管理员重置 |
| 账号被锁 | 等待30分钟自动解锁 |
| 无法登录 | 检查网络连接 |

### A.2 审批问题

| 问题 | 解决方案 |
|------|----------|
| 审批人被驳回 | 查看驳回原因，修改后重新提交 |
| 审批超时 | 联系审批人或管理员 |
| 审批流错误 | 联系管理员检查流程配置 |

### A.3 AI助手问题

| 问题 | 解决方案 |
|------|----------|
| AI不回答 | 检查API配置和配额 |
| 回答不准确 | 切换其他模型试试 |
| 响应慢 | 网络问题或高峰期 |
