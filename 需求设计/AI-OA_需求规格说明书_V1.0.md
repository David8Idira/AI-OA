> 项目名称：AI-OA（基于RuoYi的智能化OA系统）
> 文档版本：V1.1（新增打车发票对接+企业邮箱+竞品功能）
> 编制日期：2026-04-05
> 修订日期：2026-04-05
> 编制人：A1
> 状态：初稿待确认

## 一、需求概述

### 1.1 项目背景

基于RuoYi开源框架，整合OCR识别、n8n流程引擎、AI大模型能力，打造新一代智能化OA系统，实现财务审批自动化和智能报表生成。

### 1.2 核心功能范围

| 编号 | 模块 | 功能点 | 优先级 |
|------|------|--------|--------|
| F1 | 基础管理 | RuoYi原生功能继承 + 企业邮箱集成 | P0 |
| F2 | 财务审批 | OCR发票 + 打车行程单 + n8n审批流 | P0 |
| F3 | 智能报表 | 周刊/月刊/年刊自动生成 | P0 |
| F4 | AI对话 | 侧边栏AI助手+模型配置 | P0 |
| F5 | 流程中心 | n8n可视化编辑器集成 | P1 |
| F6 | 增强功能 | 竞品优秀功能（低代码+移动端+知识库） | P2 |

### 2.2 财务审批自动化模块（F2）

#### 2.2.1 OCR发票识别

| 功能点 | 描述 | 技术方案 |
|--------|------|----------|
| 发票上传 | 支持PDF/图片/扫描件 | 文件上传组件 |
| 文字识别 | 提取发票关键信息 | PaddleOCR + 阿里云OCR |
| 信息校正 | 人工校正识别结果 | 表单编辑+置信度显示 |
| 历史存档 | 原始文件+识别结果存档 | MinIO存储 |

**OCR识别字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 发票类型 | 枚举 | 增值税普票/专票/机票/火车票 |
| 发票代码 | 字符串 | 10位/12位 |
| 发票号码 | 字符串 | 8位 |
| 开票日期 | 日期 | YYYY-MM-DD |
| 购买方名称 | 字符串 | 公司名称 |
| 销售方名称 | 字符串 | 供应商名称 |
| 金额 | 小数 | 不含税金额 |
| 税额 | 小数 | 税额 |
| 价税合计 | 小数 | 含税金额 |
| 商品明细 | 列表 | 商品名称/数量/单价/金额 |

#### 2.2.2 n8n审批流程

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 报销申请 | 填写报销信息+上传附件 | 表单完整+附件上传 |
| 审批流配置 | 可视化配置审批节点 | n8n编辑器 |
| 自动审批 | 预算内+常规品类自动通过 | 规则引擎 |
| 人工审批 | 多级审批（主管→财务→CEO） | 审批/拒绝/退回 |
| 知会提醒 | 审批节点自动知会 | 站内/邮件/飞书 |
| 流程查询 | 查看申请进度/历史 | 状态时间轴 |

**审批流程模板**：

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  发起人  │───►│  主管   │───►│  财务   │───►│   CEO  │
│  提交   │    │  审批   │    │  复核   │    │  审批   │
└─────────┘    └────┬────┘    └────┬────┘    └────┬────┘
                   │              │              │
              ≤5000元        ≤20000元        >20000元
             自动通过         财务复核        CEO终审
```

#### 2.2.3 统计看板

| 功能点 | 描述 | 指标 |
|--------|------|------|
| 报销统计 | 各部门/个人报销金额 | 金额/笔数/平均周期 |

## 三、非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面响应时间 | ≤2s (P95) |
| AI响应时间 | ≤30s (文字)，≤120s (图片) |
| OCR识别时间 | ≤5s/页 |
| 并发用户数 | ≥200 |
| 数据库连接 | ≥100 |

### 3.2 安全需求

| 需求 | 说明 |
|------|------|
| 认证授权 | RBAC+JWT |
| 数据加密 | HTTPS+敏感字段AES |
| 审计日志 | 全操作记录 |
| SQL注入 | 参数化查询 |
| XSS攻击 | 输入过滤+输出编码 |
| CSRF | Token验证 |

### 3.3 兼容性需求

| 类型 | 要求 |
|------|------|
| 浏览器 | Chrome/Firefox/Safari/Edge最新2版本 |
| 移动端 | 响应式适配 |
| 操作系统 | Windows/macOS/Linux |

### 3.4 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥99.5% |
| 备份策略 | 每日全量+实时增量 |
| 灾难恢复 | RTO≤4h, RPO≤1h |

## 四、数据字典

### 4.1 核心实体

| 实体 | 说明 | 主要字段 |
