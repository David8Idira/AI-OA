## 一、项目背景与目标

### 1.1 项目背景

企业OA系统是组织内部管理的核心工具，涵盖审批流程、文档管理、协作沟通等关键业务。传统OA系统存在以下痛点：

| 痛点 | 说明 | 业务影响 |
|------|------|----------|
| 审批效率低 | 纸质/邮件审批耗时长 | 业务周期延长 |
| 报表手工制作 | 周刊/月刊等需专人整理 | 人力成本高 |
| AI能力缺失 | 无法利用大模型提效 | 竞争力不足 |
| 流程配置复杂 | n8n等工具学习成本高 | 推广困难 |

### 1.2 项目目标

**总体目标**：基于RuoYi开源框架，整合OCR识别、n8n流程引擎、AI大模型能力，打造新一代智能化OA系统。

**核心能力目标**：

```
┌─────────────────────────────────────────────────────────────┐
│                    AI-OA 核心能力                         │
├─────────────────────────────────────────────────────────────┤
│  财务审批自动化    │ OCR发票识别 → n8n流程 → 自动审批/知会     │
│  智能报表生成      │ AI生图/视频 → 周刊/月刊/年刊自动生成    │
│  AI对话助手        │ 侧边栏实时问答 → 模型配置管理            │
│  流程引擎          │ n8n工作流 → 可视化配置 → 审批流/知会流   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 财务审批自动化（核心模块）

#### 3.2.1 业务流程

```
原始票据（纸质/电子）
       ↓
   [OCR识别]
       ↓
发票信息提取（金额/税号/日期/商品）
       ↓
   [规则引擎]
       ↓
┌─────────────────────────────────────┐
│  自动审批 ←── 预算内+常规品类        │
│  人工审批 ←── 超预算/异常项          │
│  异常知会 ←── 疑似重复/格式错误      │
└─────────────────────────────────────┘
       ↓
   [n8n流程引擎]
       ↓
   审批节点（主管→财务→CEO）
       ↓
   [执行结果]
       ↓
  更新状态 → 发送通知 → 归档
```

#### 3.2.2 OCR能力要求

| 能力 | 技术选型 | 说明 |
|------|----------|------|
| 通用文字识别 | PaddleOCR / Tesseract | 通用文本 |
| 发票识别 | 阿里云OCR / 腾讯OCR | 专用票据 |
| 手写体识别 | PaddleOCR enhanced | 增强版 |
| 表格识别 | PaddleOCR + 规则 | 结构化提取 |

#### 3.2.3 n8n集成方案

```
┌──────────────────────────────────────────────────────────┐
│                    n8n Workflow Engine                    │
├──────────────────────────────────────────────────────────┤
│  Trigger: Webhook (来自AI-OA)                       │
│       ↓                                                   │
│  Node: OCR Result Parser (JSON转换)                     │
│       ↓                                                   │
│  Node: Business Rules (预算检查/品类匹配)                │
│       ↓                                                   │
│  Node: Approval Router (自动/人工/知会)                  │
│       ↓                                                   │
│  Node: Notification (飞书/邮件/站内)                     │
│       ↓                                                   │
│  Node: Callback (回写AI-OA状态)                      │
└──────────────────────────────────────────────────────────┘
```

### 3.3 智能报表生成（核心模块）

#### 3.3.1 报表周期类型

| 类型 | 生成时机 | 内容要求 | 分发方式 |
|------|----------|----------|----------|
| 周刊 | 每周五16:00 | 本周概览+数据图表 | 邮件+钉钉 |
| 月刊 | 每月最后一个工作日 | 月度总结+趋势分析 | 邮件+企业微信 |
| 季刊 | 每季度末 | 季度回顾+下季计划 | 邮件+文件 |
| 半年刊 | 每年6/12月末 | 半年度总结+表彰 | 多渠道 |
| 年刊 | 每年12月末 | 年度大盘点+来年版 | 全渠道 |

#### 3.3.2 AI生成能力

| 内容 | AI能力 | 输入 | 输出 |
|------|--------|------|------|
| 文字总结 | GPT-4/Claude/国产 | 数据+模板 | 文字内容 |
| 封面图 | DALL-E/MJ/SD | 主题描述 | 封面图片 |
| 插图 | DALL-E/SD | 内容段落 | 相关配图 |
| 视频 | Sora/Runway/国产 | 文案+图片序列 | 短视频 |
| 排版 | HTML模板引擎 | AI内容+模板 | 精美HTML/PDF |

#### 3.3.3 生成流程

```
用户配置报表需求（周期/模板/分发列表）
       ↓
   [定时触发 / 手动触发]
       ↓
   [数据采集] → 业务数据 + 外部数据
       ↓
   [AI内容生成]
       ↓
┌─────────────────────────────────────┐
│  文字总结（LLM）                    │
│  封面设计（生图模型）               │
│  数据图表（ECharts）               │
│  配图生成（可选）                   │
└─────────────────────────────────────┘
       ↓
   [模板渲染] → HTML/PDF
       ↓
   [多渠道分发] → 邮件/钉钉/飞书/企业微信
       ↓
   [归档存储] → 云盘/OSS
```
